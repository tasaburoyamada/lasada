# Lasada (ラサダ)

> **「AIにPCの全権を。ただし、隔離された『砂場』の中で。」**

Lasada（ラサダ）は、AIがあなたに代わってプログラミング、データ分析、OS操作のすべてを自律的にこなす、**Docker実行を前提とした**AIエージェント・ランタイムです。

---

## 🛡️ 「隔離されているから、自由になれる」

Lasadaの設計思想は**完全なる隔離と自由**です。

多くのAIエージェントはあなたのPC（ホストOS）を直接操作しようとしますが、それは常に破壊のリスクと隣り合わせです。Lasadaは、AIを**Dockerコンテナという「隔離された砂場」**に閉じ込めることで、そのリスクをゼロにします。

- **AIへの全権委任**: 隔離環境なので、AIに「システムを好きに書き換えていいよ」と全権を預けることができます。
- **ホストOSの絶対保護**: AIがどれだけ破壊的なコマンド（`rm -rf /`など）を実行しても、あなたの本物のPCには1ミリも影響を与えません。
- **使い捨ての実験室**: 作業が終わればコンテキスト（箱）ごと消去。常にクリーンな状態で次のタスクを始められます。

---

## 🌟 主な特徴

- **Docker-Native**: AIが動くのは常に隔離されたコンテナ内。環境を汚さず、安全に「丸投げ（--auto-run）」が可能です。
- **Tool Callingの実装**: AIとシステムの通信に「Tool Calling（関数呼び出し）」を採用。命令の崩れがなく、極めて堅牢に動作します。
- **Rust実装の超軽量コア**: 起動は一瞬。Pythonの重い依存関係に悩まされることはありません。
- **`@path` 構文**: プロンプト内で `@file.py` と打つだけで、AIの「砂場」に瞬時にコンテキストを送り込めます。

## 🚀 クイックスタート

### 1. 準備
- **Docker** がインストールされていること（必須）
- LLMのAPIキー（OpenAI / Gemini等）

### 2. セットアップ
```bash
git clone https://github.com/kubodad/lasada.git
cd lasada
./install.sh
```

### 3. 実行（砂場モード）
```bash
# AIを隔離環境（Docker）の中で起動します
lasada --sandbox
```
