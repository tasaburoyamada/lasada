# Lasada 動作前提条件

`lasada` を正常に実行・開発するために必要な環境および設定は以下の通りです。

## 1. オペレーティングシステム
- **Linux** (推奨)
- **Windows Subsystem for Linux (WSL/WSL2)**
  - 本ツールは WSL 環境をセーフなサンドボックスとして利用することを想定しています。
  - GUI 操作 (Computer Use) を行う場合は、WSLg または X サーバーの設定が必要です。

## 2. ソフトウェア・ツール
以下のツールがインストールされ、パスが通っている必要があります。

- **Rust ツールチェーン**
  - `cargo`, `rustc` がインストールされていること（Edition 2024 対応）。
- **GUI 操作・キャプチャ関連**
  - `xdotool`: マウス・キーボード操作のシミュレーションに使用。
  - `scrot` または `gnome-screenshot`: 画面キャプチャの取得に使用。
- **ドキュメント解析関連**
  - `poppler-utils` (`pdftotext`): PDF ファイルのテキスト抽出・解析に使用。
- **Python**
  - `PythonExecutor` を介したスクリプト実行を行う場合に必要。

## 3. ネットワーク・API
- **LLM 接続環境**
  - OpenAI 互換 API へのアクセス。
  - インターネット（または対象のローカルネットワーク）への疎通確認。
- **API キー**
  - 環境変数 `LLM_API_KEY` に有効なキーが設定されていること。

## 4. 知識・スキル
- **ターミナルの基本操作**
  - 環境変数の設定やコマンドの実行方法。
- **基本的セキュリティ意識**
  - AI が生成したコマンドを自動実行（`--auto-run`）する際のリスクを理解していること。
