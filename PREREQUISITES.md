# Lasada 動作前提条件

`lasada` を正常に実行・開発するために必要な環境および設定は以下の通りです。

## 1. オペレーティングシステム
- **Linux** (推奨)
- **Windows Subsystem for Linux (WSL/WSL2)**
  - 本ツールは WSL 環境をセーフなサンドボックスとして利用することを想定しています。

## 2. ソフトウェア・ツール
- **Rust ツールチェーン**
  - `cargo`, `rustc` がインストールされていること（Edition 2024 対応推奨）。
- **Bash**
  - `BashExecutor` がシステム上の `/bin/bash` を使用してコマンドを実行するため。
- **Git**
  - バージョン管理およびソースコードの取得に使用。

## 3. ネットワーク・API
- **LLM 接続環境**
  - 自社 LLM または OpenAI 互換 API へのアクセス。
  - インターネット（または対象のローカルネットワーク）への疎通確認。
- **API キー**
  - 環境変数 `LLM_API_KEY` に有効なキーが設定されていること。

## 4. 知識・スキル
- **ターミナルの基本操作**
  - 環境変数の設定やコマンドの実行方法。
- **基本的セキュリティ意識**
  - API キーの管理方法や、AI が生成したコマンドを自動実行する際のリスク理解。

## 5. 推奨されるディレクトリ構造
- プロジェクトの知見や設計思想を管理するため、上位ディレクトリに `HV-CAD-Framework/specs/` が存在し、そこに `.vlog` ファイルが蓄積されていること。
