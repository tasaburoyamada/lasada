# Lasada を使いこなそう！

Lasada は、あなたの優秀な「AIアシスタント」です。ここでは、具体的にどんな風に話しかければいいか、いくつか例を挙げて紹介します。

## 1. はじめの一歩（起動のしかた）

パソコンの画面（ターミナル）で、以下のコマンドを打つだけです。

```bash
# 隔離された「箱」の中で安全に起動
lasada --sandbox
```

起動したら、あとは普通に日本語で話しかけてください。

## 2. こんなお願いをしてみよう

### 🛠️ 「何かを作ってほしいとき」
AIはプログラムを書くのが得意です。
- 「今の時間を大きく表示する時計アプリをPythonで作って」
- 「デスクトップにある写真を、日付ごとにフォルダ分けして」

### 📊 「データをまとめてほしいとき」
面倒な事務作業も任せられます。
- 「このフォルダにあるExcelファイルを全部読み込んで、合計金額を計算して」
- 「@data.csv の中身を分析して、分かりやすいグラフにして保存して」
  *(※ `@ファイル名` と書くと、AIにそのファイルを読ませることができます)*

### 🌐 「調べ物をしてほしいとき」
ネットの情報をまとめてもらうのも便利です。
- 「最新のiPhoneのスペックを調べて、前のモデルとどこが違うか表にして」
- 「このウェブサイトのニュースを要約して」

## 3. 便利な「秘密の呪文」

Lasada をもっと便利にするためのオプションです。

- **`--auto-run`**: AIがやることをいちいち「やっていい？」と聞いてこなくなります。完全に信頼して任せるときに使います。
- **`--session <名前>`**: 前に話した内容を覚えておいてもらいたいときに、その作業に名前をつけられます。
- **`@ファイルのパス`**: プロンプトの中にこれを入れると、AIがそのファイルの中身を見てくれます（例：`このファイル @main.rs のバグを見つけて`）。

## 4. 設定のカスタマイズ

Lasada の動作（使用するモデルなど）は、設定ファイルや環境変数で自由に変更できます。

### 設定の優先順位
Lasada は以下の順番で設定を探し、最初に見つかったものを使用します：

1.  **環境変数**: `LLM_API_KEY` など、システムに直接設定した値。
2.  **プロジェクトルートの `config.toml`**: いま作業しているフォルダにある設定ファイル。
3.  **ユーザー設定 (`~/.config/lasada/config.toml`)**: **【推奨】** パソコン全体で共通して使う設定。

### ⚠️ セキュリティ上の重要な注意
プロジェクトのルートフォルダ（作業フォルダ）に `config.toml` を置く場合は、以下の点に十分注意してください：

- **APIキーの流出リスク**: `config.toml` に API キーを書き込んだまま、そのフォルダを GitHub などのインターネット上に公開（プッシュ）してしまうと、**あなたの API キーが世界中に公開され、悪用される恐れがあります。**
- **対策**: API キーは可能な限り「環境変数」として設定するか、`.gitignore` ファイルに `config.toml` を追加して、インターネットに送信されないように設定してください。

## 5. 終わるとき

作業が終わったら、`exit` と打つか、`Ctrl + C` を押せば「魔法の箱」が閉じて終了します。
