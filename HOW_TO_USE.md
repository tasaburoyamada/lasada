# Lasada 使用ガイド

このドキュメントでは、`lasada` の導入、詳細な設定、および各種機能の使い方について説明します。

## 1. クイックスタート

### インストール
リポジトリのルートで `./install.sh` を実行してください。

### 実行
```bash
lasada --model gpt-4o --session my_task
```

## 2. CLI オプション

`lasada` は起動時に様々なフラグを指定できます。

- `--debug`: デバッグ情報を有効にします。通信内容や内部の推論プロセスが標準出力に表示され、`logs/` ディレクトリに詳細なトレースログが保存されます。トラブルシューティングに最適です。
- `--model <MODEL>`: 使用する LLM モデル名を指定します（例: `gpt-4o`, `claude-3-5-sonnet`, `gemini-1.5-pro`）。`config.toml` のデフォルト設定を上書きします。
- `--session <NAME>`: セッション名を指定します。この名前で対話履歴が管理され、中断した作業を後で同じコンテキストから再開できます。
- `--auto-run`: AI が提案したコマンド（Shell, Python, GUI等）を、ユーザーの確認（[Y/n]）を待たずに即座に実行します。CI環境や自律タスクで威力を発揮します。
- `--export <PATH>`: セッション終了時に生成される Markdown レポートの保存先を指定します。デフォルトは `sessions/report_YYYYMMDD.md` です。

## 3. 高度な実行エンジン (Executor)

AI は指示に応じて、複数の実行環境を使い分けます。

### 🐚 Bash (シェルコマンド)
標準的なシステム操作、ファイル編集、Git 操作、ビルド実行などに使用します。
- **例**: `lasada` に対し「現在のディレクトリのファイルをリストアップして、src/ 配下に新しく main.rs を作成して」と指示。

### 🐍 Python 実行
複雑なデータ処理、スクレイピング、計算処理などに使用します。
- **例**: 「requests を使って https://example.com からデータを取得し、JSON形式で保存するスクリプトを書いて実行して」と指示。

### 🌐 Web & Browser (ウェブ操作)
内蔵ブラウザを使用して、情報の検索やページ解析を行います。
- **例**: 「最新の Rust 2024 エディションの新機能を調べて、要約して」と指示。

### 💻 Computer Use (GUI 操作)
画面キャプチャとマウス・キーボード操作を介して、ブラウザ以外の GUI アプリを操作します。

## 4. ファイル解析とコンテキスト管理 (@path 構文)

`lasada` のプロンプト入力において、**`@path/to/file`** 構文を使用することで、外部ファイルの情報を効率的に AI に渡すことができます。

- **ファイルの中身を読み込む**: プロンプトの中に `@src/main.rs` と記述すると、そのファイルの内容が自動的に展開され、AI がコンテキストとして参照できるようになります。
- **複数ファイルの指定**: `@src/lib.rs @tests/integration_test.rs` のように複数指定可能です。
- **インデックス登録**: 読み込まれたファイルは同時にベクトル DB にインデックスされ、将来のセッションでの RAG（検索拡張生成）に活用されます。

## 5. セッション終了とレポート

`exit` または `quit` で終了すると、そのセッションの対話内容が構造化された Markdown レポート（`sessions/report_YYYYMMDD.md`）として出力されます。これにより、作業ログの作成が容易になります。
