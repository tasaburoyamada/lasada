# Project: Lasada (Rust / HV-CAD Architecture)

Open-Interpreter を参考に、Rust による圧倒的な安定性と HV-CAD による自律的制御を融合させた、次世代のインタープリター再構築プロジェクト。

## 1. 目的
- **Python 依存の完全排除**: 複雑な依存関係、ランタイムの不安定さ、およびグローバルインタプリタロック（GIL）の制約を排除し、Rust によるネイティブな堅牢性と高パフォーマンスを実現する。
- **HV-CAD (Human-Value Centric Autonomous Development) の導入**: AI を「人格」ではなく「確率分布の操作対象」として定義し、シンボリックな状態管理（.vlog）を通じて、人間の価値判断をダイレクトに実行エンジンへ反映させる。
- **プラグイン指向の拡張性**: 実行エンジンや LLM バックエンドをトレイトベースで抽象化し、高度な安全性と柔軟な差し替え可能性を両立する。

## 2. 建築原則 (Architectural Principles)
- **事実主導 (FDOD - Fact Driven Orbit Determination)**: 推論の前に現状の「確実な事実（コード、ログ、環境）」を網羅的に調査し、ハルシネーションを構造的に抑制する。
- **ステートレス性の克服**: LLM の揮発性を補完するため、過去の判断や制約をシンボリックなベクトル形式（`.vlog`）で永続化し、コンテキストの整合性を維持する。
- **生成と評価の分離**: AI は「生成（How）」に特化し、人間は「価値判断（What is right）」を独占する。AI の出力は常に人間による正則化を受ける。

## 3. 技術目標 (Technical Goals)
- **Rust によるコア実装**: `std::process` の非同期ラップ、`tokio` による高効率なセッション管理、トレイトによるプラグイン・インターフェースの確立。
- **HV-CAD 統合**: `@BIAS`（価値基準の重み付け）や `@CONCEPT`（概念アクティベーション）を用いた、自然言語に頼らない高密度な AI 制御。
- **自律的エラー修正**: 実行エラーを事実としてフィードバックし、AI が自律的にパッチを生成・適用する「自己修復ループ」の構築。
